# Demo Asset Plan

## Screenshots Needed

1. Main cockpit showing source coverage, review queue and proof-packet value.
2. Review queue item with source authority, taxonomy match and impact rationale.
3. Alert proof packet showing reviewer approval, recipient checks and digest metadata.
4. Database unavailable state showing safe fallback behavior.

## Synthetic Input

Use bundled fixture publications and synthetic product-map data. Keep law-firm mode examples synthetic and public-source styled.

## Output To Show

Show review-ready regulatory items, product-impact scoring rationale, alert draft status, proof-packet metadata and blocked delivery until approval.

## Material To Exclude

Do not show real client product maps, private matter notes, personal recipient lists, provider tokens, Slack or Teams webhooks, Clerk secrets or alert body logs from real use.

## 60 Second Demo

1. Run fixture ingestion or open the dashboard in fixture mode.
2. Select a high-impact publication.
3. Show taxonomy match and matter/product impact.
4. Open the proof-packet or approved-alert sample.
5. Explain that external delivery has no path without review approval.

## Buyer Or Recruiter Takeaway

The app demonstrates source diligence, deterministic classification, human review gates, delivery safety and commercially useful regulatory monitoring.

## Hosting Readiness

GitHub and local demo ready with fixtures. Hosted demo should wait until auth, database retention, delivery-provider isolation and fixture-only seeding are reviewed.
